Project idea

The project idea i have come up with is An app that allows RMIT students to easily find their classes in their buildings through an Augmented reality displayed on an iPhone/android. This project will allow students and staff to view a path to their room on their mobile devices. Eg. if a student cannot find room 42 in building 12 then the app will direct a path from wherever they are standing to the selected room. Through this app, it will dramatically decrease the confusion for students trying to find new rooms/ lecture theatres as it will be on a map similar to google maps street view.

The motivation for me to come up with the idea of this project was because when i first walked into the RMIT buildings I was completely lost and had little to no idea as to where my rooms were that I had to attend. Most of the people I met and saw were in the exact same situation as I was and I still continue to see people struggling to find their rooms. The room numbers are easy but to the students, it's a ‘maze’ as some rooms require you to go into another building and cross back because elevators don't go high enough. Because of this, I thought that an app that had an AR map on it would allow students to easily find their rooms and save confusion.

The project will be based primarily for an iPhone. It will be aimed to integrate the use of AR technologys and maps to create a simple pathfinding application that will allow the user to find their room given that they are withing a set area from the building. A detailed breakdown description is as follows:

Initial primary focus:The initial focus for the project will be for students to find their way to one specific room in one building. The reason for this initial concept is because of the restraint of the timeframe for the project to be completed in. By having a small realistic goal it will allow for a higher quality application. The goal will be to have the application to allow the user to find their way to building 8/08/82, within the building 12 ground floor.

Initial idea for how paths can be founded:An idea for how paths can be formed is a camera is taken around the building taking photos of certain areas that are x distance from the last photo, these photos would be collated together and ordered. From there coding would have to be done to have the application determine the quickest route to the room. The application would then access the user's camera and draw a blue line along the floor toward the locations of the photos that were taken, photos would provide a backup for if the lines failed to work.

Or

A 3d scan would be taken for the layout of the building and the floors would be grid referenced in XYZ, this will allow for a much higher detail for tracking. From there the points would be labelled and put into an order, when a room is selected the pathfinding aspect would run a function to select which grid coordinates will lead to the door of the room. The app would then display a blue line along the ground that will lead to the room, the lines would be connected to 1x1m squares which will make the appearance of the lines jagged but will allow for a better tracking.

User useability: The project will be designed around easability of use so that new students will be able to easily navigate the application. In an attempt to do this the page design will be a large focus. The layout of instructions will have to be minimal and precise. A way this could be done is through having a ‘class search’ button so if the student does not know where their room is they can search it up easily through a database. Furthermore the AR component should be a standing out colour and clearly show a path to the room that the user needs to go to, this could be done via a line that could be customised in terms of colour and width. Or could be a dotted line showing the path.

Further opportunities:This application will provide many opportunities for expansion in the future. It could serve to replace all RMIT maps and could be further expanded to act as a semi- replacement for class timetables. The application could be expanded even further to accommodate the CBD of melbourne including which trams should be boarded. This will allow travelling much easier for new people to the city as they will be able to see a clear precise path in an AR function rather than just a map on a screen. It would go into much further detail in comparison to the already existing google AR maps. The application would prove to be much more appealing than a regular map and the useability would be greatly increased.

Software required:Some software that would be required could include, Python as a programming language, AR toolkit, a camera to map out the building's layout, a phone to test the program on and a computer to do the programming on. Python will be the best choice for a programming language as it is a widely known language and has many opportunities to reach for help if the program gets stuck, it also is a benefit to use this programing language as IT students are already learning the language. Alternatively, Javascript could be used as it also has a large user base and can meet the needs for the project. An application like AR toolkit would provide a critcal amount of tools that would aid to speed up production of the app as it has a wide availability. Such as, GPS location tracking.
